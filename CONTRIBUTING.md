### üíª Code Contributions

#### Getting Started

```bash
# Fork the repository on GitHub
# Clone your fork
git clone https://github.com/YOUR_USERNAME/fluid-stratos.git
cd fluid-stratos

# Create a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install in development mode
pip install -e .
pip install -e ".[dev]"

# Create a new branch
git checkout -b feature/your-feature-name
```

#### Code Style

- Use **clear, descriptive names** over cryptic abbreviations
- Add **comments** explaining the "why", not the "what"
- Write **docstrings** for functions and classes
- Keep functions **focused and small** (Wu Wei principle applies to code too!)

We don't enforce strict style guidelines‚Äîreadability matters more than rules.

#### Testing

While we don't have full test coverage yet, please:

1. Test your changes manually
2. Ensure existing examples still work
3. Add a new example if introducing a major feature

#### Commit Messages

Write clear commit messages:

```
Add viscosity gradient feature to FluidSTRATOS

- Implemented spatial viscosity variation
- Added get_viscosity_field() method
- Updated visualization to show viscosity map
```

## Pull Request Process

1. **Update documentation** if you changed functionality
2. **Add yourself to contributors** in README.md
3. **Describe your changes** clearly in the PR description
4. **Link related issues** using keywords like "Fixes #123"
5. Be patient and open to feedback

### PR Review Process

We aim to review PRs within 1 week. If you don't hear back:
- Politely ping the maintainer
- Join discussions on other PRs
- Keep contributing!

## Community Guidelines

### Be Kind and Respectful

- This is a **judgment-free zone**
- Ask questions‚Äîthere are no "stupid questions"
- Help others when you can
- Assume good intentions

### Embrace Beginner's Mind

- **"I don't know"** is a valid and valuable answer
- Share your learning process, not just results

### Wu Wei Collaboration

Like the Cognitive Gardener, we practice **minimal intervention, maximum effectiveness**:

- Don't over-engineer
- Don't bikeshed (argue about trivial details)
- Trust the process
- Let ideas flow naturally
